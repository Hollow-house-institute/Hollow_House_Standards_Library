• machine-readable terminology records  
• version history  

The registry acts as the Semantic Layer of the HHI Governance Stack.

---

## Terminology Stability Rules

1. Prefer existing glossary terminology whenever possible.  
2. Do not invent new governance terminology unless explicitly requested.  
3. When a glossary term is used, its meaning must remain consistent with the glossary definition.  
4. Glossary terminology functions as stable identifiers for governance concepts.  

If a definition must be modified, the assistant must flag that the change requires:

• glossary review  
• checksum update  
• version increment  

Terminology stability is mandatory.

## Behavior Expectations for AI Assistants

Assistants should:

• preserve terminology stability  
• avoid semantic drift  
• maintain cross-file consistency  
• reference canonical glossary definitions  
• reinforce terminology discoverability  

---

## Restricted Behavior

The assistant must not:

• invent new governance terminology without explicit request  
• silently modify glossary definitions  
• rename glossary identifiers  
• fragment terminology across files  
• replace canonical definitions with paraphrased variants  

---

## Constraint Violation Protocol

If a request conflicts with restricted behavior:

1. Halt generation of the conflicting content.  
2. Identify the violated rule.  
3. Provide the rule name.  
4. Request explicit authorization before proceeding.

Example:

Violation detected: attempt to invent new governance terminology.

Rule violated: NEW_TERM_POLICY = RESTRICTED

Action halted. Explicit authorization required.
